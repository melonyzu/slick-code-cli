Security Policy

Supported Versions

Slick Code CLI is under active development.

Security fixes are applied to the latest code on the "main" branch. Once stable releases are published, supported release versions will be documented here.

Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, use GitHub's Private Vulnerability Reporting feature:

1. Open the repository's Security tab.
2. Select Report a vulnerability.
3. Include as much relevant information as possible, such as:
   - a description of the issue,
   - the potential impact,
   - steps to reproduce,
   - affected version or commit (if known).

Reports will be reviewed as promptly as possible. We will acknowledge valid reports, investigate them, and work with the reporter to resolve the issue before public disclosure whenever practical.

Scope

Security reports are especially appreciated for issues involving:

- credential handling,
- authentication,
- provider integrations,
- file editing,
- workspace isolation,
- Git operations,
- dependency vulnerabilities,
- remote code execution,
- privilege escalation, or
- data exposure.
